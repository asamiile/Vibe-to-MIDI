# Product Direction

## Product

Vibe-to-MIDI is an Android-first music learning and production assistant.

It is not a traditional music theory course. It is a tool for converting intuitive musical intent into practical DAW choices.

## Learning Philosophy

Vibe-to-MIDI reverses the common music-learning flow.

Traditional flow:

```text
learn music theory -> make music
```

Vibe-to-MIDI flow:

```text
make something first -> learn why it works
```

The app should not force users to study concepts before creating sound. Users should start from vibe, emotion, movement, and sound image, then hear and touch the result before theory is introduced.

Learning should happen through making. Theory is revealed as context for a sound the user already created or heard.

## First Problem

The user has a sound image in mind, but cannot translate it into notes, scales, chords, rhythms, or sound design choices.

The app should help with this conversion:

```text
vibe / emotion / motion
-> musical traits
-> scale / chord / bass / rhythm / sound design
-> DAW-ready suggestions
```

## Audience

Primary users:

- DTM beginners and intermediate producers
- electronic music producers
- people who know the sound they want but do not know theory vocabulary
- people making club-oriented electronic music

Target music context:

- techno
- house
- ambient
- minimal
- melodic techno
- experimental electronic music

## Target Languages

Prioritize languages in this order:

1. English
2. Japanese

English should be the primary product language. Japanese support should be planned from the start, but it should not slow the first Android MVP.

## MVP Experience

One Android screen is enough for the first version.

User flow:

1. Select vibe tags.
2. Hear a short generated pattern.
3. See a visual response.
4. Get DAW-ready suggestions.

Initial vibe tags:

- dark
- floating
- tense
- repetitive
- underground
- wide
- hypnotic
- metallic
- warm
- unstable

Initial output:

- scale
- chord
- bass notes
- rhythm pattern
- sound design hint
- MIDI note list

## Product Boundaries

The app should not start as:

- a full DAW
- a streaming app
- a music theory textbook
- a full AI music generator
- a copyrighted reference-track analyzer

The first version should prove that vibe-based musical mapping is useful.
